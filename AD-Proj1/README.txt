O programa trabalha como é descrito no enunciado.

Como sugerido pelo professor, foi alterada a ordem da mensagem enviada pelo cliente.

O cliente, assim, só tem que enviar o comando e o id do recurso.
Ou seja:
	LOCK 1
	RELEASE 1
	TEST 1
	STAT 1

O id do cliente é concatenado à mensagem. 

De resto, é importante assinalar que foi incluido a serialização da mensagem.
Não era necessário para esta entrega, mas achamos que, sendo que já estava feito,
e sendo que o programa não tem erros, não havia razão para não enviar.
